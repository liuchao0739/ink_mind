## 墨智 InkMind Demo 视频脚本（约 3–5 分钟）

### 场景 1：开场 & 产品定位（约 20–30 秒）
- 画面：应用 Logo + “墨智 InkMind” 标题
- 旁白要点：
  - “欢迎来到墨智，一款用 AI 重新定义阅读体验的跨平台应用。”
  - “它可以在手机、电脑、Web、甚至鸿蒙设备上，为你提供统一的阅读体验。”

### 场景 2：书城 & 搜索（约 40 秒）
- 操作：
  - 打开首页，展示“智荐 · 为你推荐”横滑卡片
  - 向下滚动，展示“全部书库”列表
  - 在搜索框输入关键词，例如“星河”“古今”，展示即时过滤结果
- 旁白要点：
  - “首页分为智能推荐与全部书库两部分。”
  - “本次 Demo 使用的是预置示例书库，包括向四大名著致敬的《墨海古今录》和科幻向的《星河边界笔记》。”
  - “搜索完全在本地完成，支持按书名、作者和标签模糊匹配，在无后端环境也能秒级响应。”

### 场景 3：书架（约 30–40 秒）
- 操作：
  - 在书城列表中点击书签图标，将两本示例书加入书架
  - 点击右上角“书架”图标，进入书架页面
  - 展示“在读”“未阅读”状态变化
- 旁白要点：
  - “一键加书到书架，支持本地持久化保存。”
  - “书架会记录最近阅读时间，在首页和书架中都能快速找到你正在追的书。”

### 场景 4：阅读器 & 进度同步（约 60 秒）
- 操作：
  - 从首页点击《墨海古今录》，进入阅读页
  - 演示上下滚动、多段文字排版效果
  - 手动调整字体大小（如果 UI 已接好），展示阅读体验变化
  - 退出阅读器，再次进入，演示自动跳回上次阅读位置
- 旁白要点：
  - “阅读器采用长文滚动方式，更适配多端和不同屏幕尺寸。”
  - “每次滚动停止时，应用都会在本地记录章节和滚动偏移，下次打开自动恢复到上次位置。”
  - “阅读参数（如字体、行距、夜间模式等）可统一管理，为后续更深度个性化打基础。”

### 场景 5：AI 书摘 —— 智记（约 40 秒）
- 操作：
  - 在阅读器右上角点击“书摘”按钮
  - 展示底部弹窗：章节摘要 + 若干关键句
  - 在另一章节演示基于规则的自动抽取效果
- 旁白要点：
  - “智记会为每一章生成 ‘AI 书摘’。”
  - “对于示例书，我们预置了高质量摘要；同时也实现了本地规则抽取算法，基于关键词和句长打分选出候选金句。”
  - “未来可以无缝切换到云端大模型生成更深度的阅读理解。”

### 场景 6：AI 朗读 —— 智声（约 30–40 秒）
- 操作：
  - 在同一页面点击“朗读”按钮，开始朗读当前章节
  - 展示暂停/继续或语速调节（根据实现程度展示）
- 旁白要点：
  - “点击右上角的扬声器图标，系统会直接朗读当前章节内容。”
  - “朗读基于 `flutter_tts` 封装，调用系统 TTS 引擎，保持全端纯前端实现。”
  - “这对视力疲劳、通勤途中想听书的用户尤其实用。”

### 场景 7：阅读画像（约 40–50 秒）
- 操作：
  - 返回首页，点击右上角“阅读画像”图标
  - 展示最近 14 天阅读时长的柱状图
  - 演示顶部摘要文案，如“最近 7 天你累计阅读 XX 分钟，最常看的类别是 XX”
- 旁白要点：
  - “每次进入阅读器都会自动记录本次阅读时长，并按日汇总。”
  - “这里展示了最近的阅读轨迹，帮助用户看到自己的 ‘阅读曲线’。”
  - “偏好题材分析可以指导智荐模块做更精准的推荐，也为未来的阅读计划和成长体系提供基础数据。”

### 场景 8：多端展示（可穿插或单独一段，约 30 秒）
- 操作：
  - 切换到 Web 浏览器，打开 `build/web` 部署后的页面，快速展示同样的首页和阅读器
  - 如有条件，可简单拍摄 Android / iOS / 桌面端运行画面拼接
- 旁白要点：
  - “依托 Flutter，一套代码即可覆盖 Android / iOS / 鸿蒙 / Web / 桌面多端。”
  - “通过 FVM 管理多版本 Flutter SDK，适配了鸿蒙定制版本 3.22.1-ohos-1.0.7。”
  - “这使得墨智可以真正做到 ‘智阅随身’——无论用户在哪个屏幕上打开，都能延续阅读体验。”

### 场景 9：收尾（约 20–30 秒）
- 画面：回到应用 Logo 或阅读器温暖画面
- 旁白要点：
  - “墨智 InkMind 目前已经实现了书城、书架、阅读器、AI 书摘、AI 朗读和阅读画像等完整闭环。”
  - “在保持纯前端、无后端依赖的前提下，为未来接入大模型和丰富内容生态预留了充分空间。”
  - “期待在谷医堂 AI 大赛中，与大家一起探索 AI + 阅读的更多可能。”
