## 墨智 InkMind · PPT 大纲

### 1. 封面
- **标题**：墨智 InkMind —— AI 多端智能阅读体验
- **副标题**：谷医堂首届 AI 大赛参赛作品
- **信息**：作者、日期、版本号
- **视觉**：墨迹 + 芯片/脑波元素，突出“墨 + 智”

### 2. 背景与痛点
- 碎片化时代的阅读困难：注意力涣散、阅读连续性差、多设备切换不顺畅
- 传统阅读 App 的局限：推荐千人一面、阅读行为数据难以被“看见”
- AI 浪潮下的新机会：个性化推荐、智能朗读、行为画像驱动长期陪伴

### 3. 产品定位与目标
- **定位**：面向爱看长篇小说/公版经典的用户，用 AI 提供“更懂你”的跨设备阅读伴侣
- **目标**：
  - 读得下去：舒适的阅读器 + 智声朗读
  - 读得聪明：智荐推荐 + 智记书摘
  - 看得见成长：阅读时长、偏好题材等行为画像

### 3.1 命名与品牌故事（单独一页）
- **中文名：墨智**  
  - “墨”代表书籍、文字与墨水，象征深厚的阅读底蕴和知识积累；  
  - “智”代表人工智能（AI），对应智能推荐、个性化阅读、语音合成、行为分析等能力。
- **英文名：InkMind**  
  - 直译为“墨水思维”，强调“以文字为介质，用 AI 打开思维边界”；  
  - 中英文读音和谐（墨智 Mò Zhì / InkMind /ɪŋk maɪnd/），便于品牌传播。
- **AI 子品牌**：  
  - **智荐**：AI 推荐算法模块；  
  - **智声**：AI 朗读模块；  
  - **智记**：AI 书摘与摘要模块。  
  将三个子能力统一在“墨智”品牌之下，方便在页面与运营物料中反复强化记忆点。

### 4. 产品整体架构
- 一图展示（可用 Mermaid 导出的架构图截图）：
  - 前端：Flutter + Riverpod
  - 数据层：本地 JSON 书库 + Hive 本地存储
  - AI 模块：规则推荐引擎、书摘服务、TTS 封装
  - 多端适配：Android / iOS / 鸿蒙 / Web / Windows / macOS

### 5. 核心功能展示（含截图）
- **书城 & 搜索**
  - 示例书库（墨海古今录 / 星河边界笔记等）
  - 按书名 / 作者 / 标签本地搜索
- **书架**
  - 一键加入/移除
  - 最近阅读自动置顶
- **阅读器**
  - 长文滚动、夜间模式预留、字体大小/行距调整（可在 Demo 中展示字体放大效果）
  - 退出后自动恢复阅读进度

### 6. AI 能力亮点
- **智荐 Recommendation**
  - 规则：基于阅读时间 + 最近阅读类别 + 标签 + 热度，计算综合得分
  - 页面：首页“智荐 · 为你推荐”横滑卡片
  - 价值：在无后端/无大模型的前提下，仍然给出“看起来懂你”的推荐结果
- **智记 Highlights**
  - 预置 AI 书摘：为示例书的关键章节人工/半自动写摘要与金句
  - 规则抽取：分句 + 关键词 + 句长打分，自动挑选 2–3 句高亮
  - 展示形式：阅读器右上角“书摘”按钮，底部弹窗展示摘要 + 关键句
- **智声 TTS**
  - 基于 `flutter_tts`，调用系统 TTS 引擎
  - 一键朗读当前章节，支持暂停/停止、语速调节（在 Demo 中预留设置入口）

### 7. 阅读行为画像
- 数据采集：
  - 每次进入阅读器记录开始时间，退出时统计本次时长
  - 按日聚合阅读分钟数、书本阅读次数、类别
- 可视化：
  - 最近 14 天阅读时长柱状图（使用 `fl_chart`）
  - 自动生成文案：“你最近更偏爱 XX 类题材，平均每天阅读 XX 分钟”

### 8. 多端展示
- 截图组合：
  - Android 手机：书城 + 阅读器
  - iPhone：书架 + AI 书摘
  - Web 浏览器：首页推荐 + 阅读画像
  - 桌面端：左右分栏（书架 + 阅读）
- 强调：同一套 Flutter 代码，通过 FVM 管理版本，轻量适配多端（现已支持 Android / iOS / 鸿蒙（示意）/ Web / Windows / macOS，未来可平滑扩展到小程序等形态，实现“智阅随身”）

### 9. 技术实现亮点
- FVM 管理多 Flutter 版本，兼容鸿蒙定制 SDK
- Riverpod 用于状态管理（书架、阅读器、AI 模块、统计）
- Hive 本地持久化（离线可用、无后端依赖）
- 可扩展性设计：Book / Chapter / RecommendationEngine / StatsRepository 等均预留字段，可接入云端大模型或后端服务

### 10. 未来规划
- 真正接入大模型：
  - 用 LLM 做章节级摘要、人物关系图谱、主题分析
  - 使用向量检索实现“在书中搜索知识点”
- 更多内容来源：
  - 支持 EPUB、本地导入 TXT 自动分章
  - 接入合法版权内容源
- 社交与社群：
  - 书评、读书进度分享、读书圈运营玩法

### 11. 总结与呼应
- 再次强调名字内涵：“墨”代表阅读底蕴，“智”代表 AI 赋能
- 形成闭环：
  - **AI + 阅读体验 + 多端兼容 + 无后端可落地 Demo**
- 呼应谷医堂 AI 大赛主题：展示如何用 AI 让数字内容消费更高效、更贴心
